\section{Introduction aux contrats}

\begin{overviewbox}
  En s'appuyant sur la leçon précédente (\emph{Algorithmes pilotés par les données}), les élèves découvrent les fonctions Pyret pour travailler avec des tables~: \emph{trier}, \emph{raccourcir}, \emph{visualiser} et \emph{extraire} des données. Ils sont introduits aux \textbf{contrats} comme «~mode d'emploi des fonctions~» (nom, domaine, image).
\end{overviewbox}

% ---------- Launch ----------
\subsection{Lancement}

\img{images/screenshot-texto.png}{width=0.35\textwidth}{Capture d'une application de messagerie proposant des alternatives pour un mot possiblement mal orthographié.}

\begin{activitybox}
  Assurez-vous d'avoir ouvert le fichier \textbf{Correcteur-Orthographique-Starter.arr} dans l'éditeur Pyret, et que votre \emph{Zone de Définitions} contient~:

  \begin{center}
    \texttt{resultats = alt-words("plation", WORDS-L-FR, 4)}
  \end{center}

  Cliquez sur «~Run~», puis évaluez \texttt{resultats} dans la Zone d'Interactions~: vous devez voir une table avec deux colonnes, \texttt{word} et \texttt{edit-distance}, contenant les suggestions de correction orthographique de \texttt{"plation"} à distance d'édition $\leq 4$ selon le dictionnaire français \texttt{WORDS-L-FR} (40~000~mots).
\end{activitybox}

Notre correcteur nous donne beaucoup de données à explorer, sous forme d'une \textbf{table} de mots suggérés et de leurs distances d'édition. Nous savons déjà sauvegarder cette table dans une variable (\texttt{resultats}) pour éviter de la recalculer à chaque fois.

Mais peut-on faire quelque chose d'\emph{intéressant} avec ces données~?

\begin{thinkbox}
  Un correcteur ne nous montre pas une table de 500~suggestions. Notre téléphone ne nous affiche pas la distance d'édition de chaque mot proposé. Mais ce n'est pas parce que la distance d'édition n'est pas affichée qu'elle n'est pas utilisée~: le téléphone s'en sert pour choisir les mots \emph{les plus probables} à suggérer.

  \begin{itemize}
    \item Si vous avez un téléphone sous la main, essayez de taper «~plation~» dans une nouvelle conversation. Sinon, ouvrez un document texte et tapez «~plation~».
    \item Quelles orthographes alternatives votre appareil propose-t-il~?
  \end{itemize}

  Nous voulons prendre le résultat de \texttt{alt-words} et \emph{lui faire quelque chose}, exactement comme le font les téléphones et les correcteurs.
\end{thinkbox}

% ---------- Investigate: Trier et raccourcir ----------
\subsection{Trier et raccourcir les tables}

\subsubsection{Trier les résultats}

Les erreurs à 1~typo sont plus fréquentes que les erreurs à 2 ou 3~typos. Nous aimerions donc \textbf{trier} les suggestions, en mettant les plus probables en haut.

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Testez ces deux expressions dans la Zone d'Interactions et notez ce que vous observez~:
    \begin{lstlisting}
sort(resultats, "edit-distance", false)
sort(resultats, "word",         true)
    \end{lstlisting}
    \item Vous avez déjà vu les types \texttt{Number}, \texttt{String} et \texttt{Boolean}. Quel type de données la fonction \texttt{sort} produit-elle~?
    \item La fonction \texttt{sort} prend trois arguments. Décrivez-les dans un tableau (type + information contenue).
  \end{enumerate}
\end{activitybox}

Il existe beaucoup d'autres fonctions dans Pyret. On les décrit à l'aide de \textbf{contrats}. Le contrat de \texttt{sort} est~:

\contrat{\# sort :: (Table t, String colonne, Boolean croissant) -> Table}

Chaque contrat contient trois éléments~:
\begin{enumerate}[leftmargin=*]
  \item \textbf{Nom} — le nom de la fonction, que l'on écrit chaque fois qu'on veut l'utiliser.
  \item \textbf{Domaine} — le(s) type(s) de données que la fonction \emph{consomme} (ses arguments).
  \item \textbf{Image} (\emph{Range}) — le type de donnée que la fonction \emph{produit}.
\end{enumerate}

Les contrats sont le \emph{mode d'emploi} des fonctions.

\begin{activitybox}
  \begin{enumerate}[leftmargin=*, resume]
    \item Quel code permet de trier \texttt{resultats} par ordre alphabétique, de A à Z~?
  \end{enumerate}
\end{activitybox}

\subsubsection{Raccourcir les résultats}

Quand on fait une faute de frappe, notre téléphone ne nous affiche que 3 à 5~suggestions. Nous aimerions \textbf{raccourcir} la table pour ne garder que les premières lignes.

\begin{activitybox}
  \begin{enumerate}[leftmargin=*, resume]
    \item Tapez \texttt{first\allowbreak-n\allowbreak-rows(resultats, 10)}. Que se passe-t-il~?
    \item Si nous voulions une table des 5~premières lignes de \texttt{resultats}, quel code écririons-nous~?
    \item Combien d'arguments composent le \emph{Domaine} de \texttt{first-n-rows}~? Quel est son \emph{Image}~?
    \item En vous inspirant du contrat de \texttt{sort}, écrivez le contrat de \texttt{first-n-rows}~:

    \begin{center}
      \texttt{nom\_fonction :: Domaine -> Image}
    \end{center}
    \item Pyret possède aussi une fonction \texttt{last-n-rows}. \textbf{Faites une prédiction}~: quel est son contrat~?
    \item Testez-la dans Pyret~! Votre prédiction était-elle juste~?
  \end{enumerate}
\end{activitybox}

% ---------- Investigate: Visualiser ----------
\subsection{Visualiser les tables}

\subsubsection{Tables de fréquence, diagrammes en barres et camemberts}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Faites défiler tous les mots suggérés dans \texttt{resultats}. Quelle \texttt{edit-distance} est la plus fréquente~? La moins fréquente~?
    \item Évaluez \texttt{count(resultats, "edit-distance")}. Que renvoie Pyret~?
    \item Écrivez le contrat de la fonction \texttt{count}~:
    \begin{center}
      \texttt{nom\_fonction :: Domaine -> Image}
    \end{center}
  \end{enumerate}
\end{activitybox}

Le contrat de la fonction \texttt{bar-chart} est~:

\contrat{\# bar-chart :: Table, String -> Image}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*, resume]
    \item Réalisez un diagramme en barres de votre table \texttt{resultats}, en utilisant \texttt{"edit-distance"} comme colonne. Qu'obtenez-vous~?
    \item Placez votre curseur sur l'une des barres. Quelle information cela vous donne-t-il~?
    \item Comment cette information se relie-t-elle à la table renvoyée par \texttt{count}~?
    \item Essayez de faire un diagramme en barres utilisant la colonne \texttt{"word"}. Cette visualisation est-elle \emph{utile}~? Pourquoi~?
  \end{enumerate}
\end{activitybox}

Le contrat de la fonction \texttt{pie-chart} est~:

\contrat{\# pie-chart :: Table, String -> Image}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*, resume]
    \item Réalisez un \emph{camembert} de votre table \texttt{resultats}, en utilisant \texttt{"edit-distance"} comme colonne.
  \end{enumerate}
\end{activitybox}

\subsubsection{Les vrais correcteurs renvoient des mots, pas des tables~!}

Définissons une ligne. En bas de la Zone de Définitions, tapez~:

\begin{lstlisting}
ligne2 = row-n(resultats, 1)
\end{lstlisting}

Cliquez sur «~Run~».

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Évaluez \texttt{ligne2} dans la Zone d'Interactions. Qu'obtenez-vous~?
    \item Avec vos propres mots, expliquez ce que fait \texttt{row-n}.
    \item Complétez le contrat de \texttt{row-n} (l'Image est donné~: \texttt{Row}).
    \begin{center}
      \texttt{row-n :: Domaine -> Row}
    \end{center}
    \item Que renvoient \texttt{ligne2["word"]} et \texttt{ligne2["edit-distance"]}~?
    \item Évaluez \texttt{row-n(resultats, 1)["word"]}. Qu'obtenez-vous~?
  \end{enumerate}
\end{activitybox}

Un code comme \texttt{ligne2["word"]} ou \texttt{ligne2["edit-distance"]} s'appelle un \textbf{accesseur de ligne} (\emph{row accessor}). Il permet d'extraire la valeur d'une cellule précise de la table.

\begin{activitybox}
  \begin{enumerate}[leftmargin=*, resume]
    \item Avec vos propres mots, que font les accesseurs de ligne~?
    \item Quel code produit le \emph{premier} mot suggéré par la table~?
    \item Comment obtenir la \texttt{edit-distance} du \emph{cinquième} mot suggéré par la table~?
  \end{enumerate}
\end{activitybox}

\begin{instructornotes}
  À ce stade, les élèves devraient être à l'aise avec~:
  \texttt{row-n}\,$\to$\,\texttt{Row},\; \texttt{row["colonne"]}\,$\to$\,valeur de cette cellule.
  Bien insister sur la numérotation qui commence à \textbf{0}~: \texttt{row-n(t, 0)} renvoie la \emph{première} ligne.
\end{instructornotes}

% ---------- Synthesize ----------
\subsection{Synthèse}

\begin{thinkbox}
  \begin{itemize}
    \item \texttt{first-n-rows} et \texttt{last-n-rows} ont \emph{exactement} le même Domaine et la même Image, mais font des choses différentes. Il en va de même pour \texttt{circle} et \texttt{triangle}. Comment est-il possible d'avoir le même Domaine \emph{et} la même Image, tout en se comportant si différemment~?
    \begin{itemize}
      \item Ce qu'une fonction a \textbf{besoin} n'est pas ce qu'elle \textbf{fait}.
    \end{itemize}
    \item Pourquoi \texttt{sort} a-t-il besoin de plus d'arguments dans son Domaine que les deux autres~?
    \begin{itemize}
      \item Parce qu'elle a besoin de \emph{plus d'informations}~: la colonne et l'ordre.
    \end{itemize}
    \item Pourquoi une application de correction orthographique aurait-elle besoin d'utiliser des accesseurs de ligne~?
    \begin{itemize}
      \item Parce qu'on veut suggérer \emph{un mot}, pas toute une ligne de la table~!
      \item Et on peut vouloir tenir compte de la \texttt{edit-distance} pour faire cette suggestion.
    \end{itemize}
  \end{itemize}
\end{thinkbox}

\begin{keypointbox}
  \textbf{Contrat}~: le mode d'emploi d'une fonction.
  \[
  \text{nom} \;::\; \text{Domaine} \;\to\; \text{Image}
  \]
  Le \textbf{Domaine} décrit ce que la fonction consomme. L'\textbf{Image} décrit ce qu'elle produit. Deux fonctions peuvent avoir le même contrat et faire des choses différentes~: le contrat dit \emph{quoi}, pas \emph{comment}.
\end{keypointbox}

\section{Composer avec les Cercles d'Évaluation}

\begin{overviewbox}
  Les élèves apprennent à combiner plusieurs fonctions en un seul programme, en utilisant les \textbf{Cercles d'Évaluation}~: une représentation visuelle de la structure de l'expression, indépendante de la syntaxe.
\end{overviewbox}

% ---------- Launch ----------
\subsection{Lancement}

Votre téléphone ne vous montre pas toutes les suggestions possibles~:
\begin{itemize}
  \item il n'en affiche que les 3 à 5~premières~;
  \item il les trie en général par \texttt{edit-distance} croissante (les erreurs les plus petites d'abord).
\end{itemize}

\begin{thinkbox}
  Comment pourrions-nous combiner les fonctions que nous avons déjà vues pour reproduire ce comportement~? Discutez-en avec vos voisin·es.
\end{thinkbox}

Si nous voulons \emph{trier} \emph{et} raccourcir notre table~:
\begin{itemize}
  \item En regardant l'\textbf{Image} de \texttt{sort}, on sait qu'elle produit une \texttt{Table}~:
  \contrat{\# sort :: (Table t, String colonne, Boolean croissant) -> Table}
  \item En regardant le \textbf{Domaine} de \texttt{first-n-rows}, on sait qu'elle \emph{consomme} une \texttt{Table}~:
  \contrat{\# first-n-rows :: (Table t, Number n) -> Table}
\end{itemize}

\noindent Mais comment faire travailler \texttt{sort} et \texttt{first-n-rows} \emph{ensemble}~?

% ---------- Investigate: Cercles d'Évaluation ----------
\subsection{Cercles d'Évaluation}

Une façon d'organiser nos pensées est de \textbf{diagrammer} ce que nous voulons faire. Les \textbf{Cercles d'Évaluation} nous aident à réfléchir à \emph{quoi} faire, sans nous soucier de la syntaxe (guillemets, parenthèses, etc.). Ils nous permettent d'utiliser tout notre cerveau pour \emph{réfléchir}, avant de l'utiliser pour \emph{coder}.

Les règles sont simples~:
\begin{enumerate}[leftmargin=*]
  \item Chaque Cercle contient \textbf{une~—~et une seule~—} fonction, écrite en haut.
  \item Les arguments de la fonction sont écrits de gauche à droite, au milieu du Cercle.
  \item Les valeurs (Nombres, Strings, Booléens) sont écrites telles quelles. Ce n'est que lorsqu'on veut \emph{utiliser une fonction} que l'on dessine un Cercle imbriqué.
  \item Les Cercles peuvent contenir d'autres Cercles~!
\end{enumerate}

\subsubsection{Observer et s'interroger}

Supposons que nous voulons voir les 5~premiers résultats de \texttt{resultats}, triés par \texttt{edit-distance} croissante. Commençons par générer la table triée, puis utilisons \texttt{first\allowbreak-n\allowbreak-rows} pour prendre les 5~premières.

\begin{center}
  \begin{tabular}{@{}c@{}}
    \cercle{sort}{resultats, "edit\allowbreak-distance", true}%
    \hspace{1em}$\to$\hspace{1em}%
    \cercleV{first-n-rows}{\cercle{sort}{resultats, "edit\allowbreak-distance", true}}{, 5}
  \end{tabular}
\end{center}

En code Pyret~:
\begin{lstlisting}
sort(resultats, "edit-distance", true)
first-n-rows(sort(resultats, "edit-distance", true), 5)
\end{lstlisting}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Que \textbf{remarquez}-vous~?
    \item Que \textbf{vous demandez}-vous~?
  \end{enumerate}
\end{activitybox}

Pour convertir un Cercle d'Évaluation en code, on part de l'\emph{extérieur} et on va vers l'\emph{intérieur}. Après le nom de chaque fonction, on ouvre une paire de parenthèses, puis on convertit chaque argument contenu dans le Cercle.

\subsubsection{À vous de jouer}

Supposons que vous vouliez que l'ordinateur prenne les 200~premières lignes de \texttt{resultats}, puis les trie par \texttt{edit-distance} \emph{décroissante}.

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item \textbf{Prendre les 200 premières lignes}~: dessinez le Cercle d'Évaluation correspondant.
    \item \textbf{Trier par edit-distance décroissante}~: dessinez le Cercle d'Évaluation qui prend le précédent en argument.
    \item Convertissez le premier Cercle en code Pyret.
    \item Convertissez le second Cercle en code Pyret.
  \end{enumerate}
\end{activitybox}

\begin{instructornotes}
  Insister sur la \emph{lecture} du Cercle~: on commence par la fonction la plus extérieure, qui consomme le résultat de la fonction intérieure. C'est exactement la structure d'une expression imbriquée en mathématiques~: $f(g(x))$.
\end{instructornotes}

% ---------- Investigate: Interface du correcteur ----------
\subsection{Interface du correcteur orthographique}

Pyret connaît bien plus que des \texttt{Number}, \texttt{String}, \texttt{Boolean} et \texttt{Table}~: il sait aussi produire des \textbf{Images}. Nous allons construire une petite interface visuelle pour notre correcteur.

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Quels sont les deux \emph{premiers} mots de votre table \texttt{resultats}~?
    \item Quel code permet d'extraire la colonne \texttt{"word"} de la \emph{première ligne} de la table~?
  \end{enumerate}
\end{activitybox}

Voici quelques contrats (et exemples~!) pour de nouvelles fonctions~:

\begin{center}
\small
\begin{tabularx}{\textwidth}{l X}
  \toprule
  \textbf{Contrat} & \textbf{Exemple} \\
  \midrule
  \texttt{string-length :: String -> Number} & \texttt{string-length("bonjour")} $\to$ 7 \\
  \texttt{text :: (String, Number, String)} \texttt{-> Image} & \texttt{text("IA~!", 50, "blue")} \\
  \bottomrule
\end{tabularx}
\end{center}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*, resume]
    \item \texttt{string-length("bonjour")} renvoie~? Que fait \texttt{string-length}~? \emph{Elle compte le nombre de caractères d'une chaîne.}
    \item \textbf{Prédisez avant de taper}~: pour chaque expression ci-dessous, que pensez-vous que Pyret va renvoyer~? Tapez-les ensuite pour vérifier.
    \begin{lstlisting}
string-length(row-n(resultats, 1)["word"])
string-length(row-n(resultats, 2)["word"])
    \end{lstlisting}
    \item Évaluez \texttt{text("IA~!", 50, "blue")}. Qu'obtenez-vous~?
    \item Quel code permet d'afficher le mot de la \emph{deuxième ligne} de \texttt{resultats} en grandes lettres bleues~?
  \end{enumerate}
\end{activitybox}

La plupart des fonctions ci-dessous ont leur contrat \emph{incomplet}~: à vous d'expérimenter dans Pyret pour compléter les parties manquantes~!

\begin{center}
\small
\begin{tabularx}{\textwidth}{l l l}
  \toprule
  \textbf{Nom} & \textbf{Domaine} & \textbf{Image} \\
  \midrule
  \texttt{triangle}        & \texttt{Number, String, String} & \texttt{Image} \\

  \textbf{À vous de créer~!} Utilisez les contrats ci-dessus pour ajouter de la couleur, plusieurs formes (et peut-être même plusieurs suggestions~!) à votre interface.
\end{keypointbox}

\begin{instructornotes}
  L'exploration des images est une activité à \emph{seuil bas / plafond haut} qui devrait intéresser tout le monde. Certains élèves trouveront \emph{tous} les contrats, la plupart ne le feront pas. Dès que le premier élève a réussi à utiliser la fonction \texttt{text}, donnez des indications sur les fonctions à prioriser avec le temps restant.

  Ne pas essayer de garder tous les élèves au même rythme. Ce qui compte, c'est que \emph{chacun·e} ait l'occasion d'explorer.
\end{instructornotes}

\subsection{Synthèse}

\begin{thinkbox}
  \begin{itemize}
    \item Pourquoi peut-il être utile de séparer les étapes de \emph{réflexion} et de \emph{codage}~? Pourquoi ne pas tout faire en même temps, tout le temps~?
    \begin{itemize}
      \item Les Cercles nous libèrent de la syntaxe pour se concentrer sur la \emph{structure}. Une fois la structure claire, la traduction en code est mécanique.
    \end{itemize}
  \end{itemize}
\end{thinkbox}

\begin{keypointbox}
  \textbf{Composer} des fonctions, c'est utiliser le \emph{résultat} d'une fonction comme \emph{argument} d'une autre. Les Cercles d'Évaluation sont la notation visuelle de cette composition~: le Cercle intérieur produit la valeur que consomme le Cercle extérieur.
\end{keypointbox}

\section*{Annexe B~: Exemple français «~plation~»}

Dans la leçon originale, on utilise \texttt{alt-words("plation", WORDS-L, 4)} sur le dictionnaire anglais pour obtenir une riche table de suggestions. Dans notre adaptation française, \texttt{"plation"} est évalué contre \texttt{WORDS-L-FR} (40~000~mots français)~; la table \texttt{resultats} contiendra donc les suggestions les plus proches selon le dictionnaire français, par exemple~:

\begin{itemize}
  \item À distance 1~: \texttt{plation} lui-même s'il apparaît dans le dico, sinon très peu de mots (peut-être \texttt{"platon"}, prénom du philosophe).
  \item À distance 2--3~: des mots comme \texttt{"platine"}, \texttt{"positions"}, \texttt{"ration"}, etc.
  \item À distance 4~: une explosion combinatoire, d'où l'intérêt de \emph{trier} puis \emph{raccourcir} pour ne garder que les suggestions les plus pertinentes.
\end{itemize}

Si vous voulez expérimenter avec des mots français plus familiers, essayez~:
\begin{lstlisting}
resultats = alt-words("maizon",    WORDS-L-FR, 3)
resultats = alt-words("ortograf",  WORDS-L-FR, 4)
resultats = alt-words("ecole",     WORDS-L-FR, 1)
\end{lstlisting}
