\section{Introduction}

\begin{overviewbox}
  En s'appuyant sur les types de données simples, les élèves découvrent les fonctions Pyret pour travailler avec des tables, créer des visualisations et des images. Ils sont initiés aux \textbf{contrats} comme «~mode d'emploi des fonctions~» et aux \textbf{Cercles d'Évaluation} pour visualiser la composition de fonctions.
\end{overviewbox}

\subsection{Lancement}

\img{images/screenshot-texto.png}{width=0.3\textwidth}{Capture d'une appli de messagerie proposant des alternatives}

\begin{activitybox}
  Assurez-vous d'avoir ouvert le fichier \texttt{Correcteur-Orthographique-Starter.arr} et que votre Zone de Définitions contient~:

  \begin{center}
    \pyret{resultats = alt-words("plation", WORDS-XS, 4)}
  \end{center}

  Cliquez sur «~Run~».
\end{activitybox}

Notre correcteur nous donne des données sous forme de \textbf{tables} de mots suggérés avec leurs distances d'édition. On peut sauvegarder ces données avec des définitions de variables.

\begin{thinkbox}
  \textbf{Peut-on faire quelque chose d'\emph{intéressant} avec ces données~?}
\end{thinkbox}

Un correcteur ne montre pas 500 suggestions, ni la distance d'édition. Mais il les \emph{utilise} en interne pour sélectionner les mots les plus probables.

\begin{activitybox}
  Essayez de taper \pyret{"plation"} dans un nouveau message texte ou document. Quelles alternatives votre appareil suggère-t-il~?
\end{activitybox}

\section{Trier et réduire les tables}

\subsection{Découvrir \pyret{sort}}

\begin{codeexample}
  Dans la Zone d'Interactions~:
  \begin{itemize}[leftmargin=*]
    \item \pyret{sort(resultats, "edit-distance", true)}
    \item \pyret{sort(resultats, "edit-distance", false)}
    \item \pyret{sort(resultats, "word", true)}
  \end{itemize}
\end{codeexample}

La fonction \pyret{sort} prend trois \textbf{arguments}~:

\begin{center}
\begin{tabular}{lll}
  \toprule
  \textbf{Argument} & \textbf{Type} & \textbf{Ce qu'il contient} \\
  \midrule
  1 & Table  & La table à trier \\
  2 & String & La colonne sur laquelle trier \\
  3 & Boolean & \pyret{true} = croissant, \pyret{false} = décroissant \\
  \bottomrule
\end{tabular}
\end{center}

\subsection{Les contrats}

Chaque \textbf{contrat} a trois parties~:
\begin{enumerate}[leftmargin=*]
  \item \textbf{Nom} — le nom de la fonction
  \item \textbf{Domaine} — le(s) type(s) des données qu'on donne
  \item \textbf{Image} — le type de donnée produite
\end{enumerate}

\begin{keypointbox}
  Les contrats sont le \textbf{mode d'emploi} des fonctions. Le contrat de \pyret{sort} s'écrit~:

  \contrat{sort :: Table, String, Boolean -> Table}

  On peut aussi annoter le Domaine avec des noms de variables~:

  \pyret{sort :: (t :: Table, column :: String, ascending :: Boolean) -> Table}
\end{keypointbox}

\subsection{Découvrir \pyret{first-n-rows}}

\contrat{first-n-rows :: Table, Number -> Table}

\begin{codeexample}
  \begin{itemize}[leftmargin=*]
    \item \pyret{first-n-rows(resultats, 5)}
    \item \pyret{first-n-rows(sort(resultats, "edit-distance", true), 10)}
  \end{itemize}
\end{codeexample}

\begin{activitybox}
  Complétez la section \textbf{Trier et réduire les tables} du cahier d'exercices.
\end{activitybox}

\begin{thinkbox}
  Peu de suggestions ont une \pyret{edit-distance} de 1, mais beaucoup plus aux distances 2, 3 et 4. La croissance est presque exponentielle~! Comment communiquer cela sans faire parcourir toute la table~?
  \begin{itemize}
    \item Compter les suggestions par distance
    \item Faire un graphique
  \end{itemize}
\end{thinkbox}

\section{Composer avec les Cercles d'Évaluation}

\begin{overviewbox}
  Maintenant qu'on sait trier et réduire des tables, on peut \textbf{combiner} ces fonctions pour obtenir exactement ce qu'on veut. Les \textbf{Cercles d'Évaluation} (\emph{Circles of Evaluation}) sont l'outil visuel utilisé par Bootstrap pour représenter la composition de fonctions~: chaque cercle contient une fonction et ses arguments, et les cercles s'imbriquent.
\end{overviewbox}

\subsection{Pourquoi composer~?}

Le tri seul donne une table de 500 suggestions classées. Ce qu'on veut, c'est \textbf{les 5 meilleures} — les plus proches du mot tapé. Pour ça, on a besoin de \emph{deux} fonctions~:

\begin{enumerate}[leftmargin=*]
  \item \pyret{sort(resultats, "edit-distance", true)} — trier par distance croissante
  \item \pyret{first-n-rows(..., 5)} — garder les 5 premiers
\end{enumerate}

La composition, c'est mettre la sortie d'une fonction dans l'entrée d'une autre.

\subsection{Du Cercle au code}

Voici comment on représente \pyret{first-n-rows(sort(resultats, "edit-distance", true), 5)} avec un Cercle d'Évaluation~:

\begin{center}
\fbox{%
\begin{minipage}{0.85\textwidth}
  \centering
  \textbf{Le cercle extérieur} (premier dans le code)~: \pyret{first-n-rows}

  \vspace{0.3em}
  \qquad \textbf{Son 1\ier{} argument} est lui-même un cercle~: \pyret{sort}

  \vspace{0.3em}
  \qquad\qquad \textbf{Ses arguments}~: \pyret{resultats}, \pyret{"edit-distance"}, \pyret{true}

  \vspace{0.3em}
  \qquad \textbf{Son 2\ieme{} argument}~: \pyret{5}

  \vspace{1em}
  $\rightarrow$ \quad \pyret{first-n-rows(sort(resultats, "edit-distance", true), 5)}
\end{minipage}%
}
\end{center}

\begin{keypointbox}
  La fonction \textbf{la plus à l'extérieur} du cercle est la \textbf{première} dans le code. On lit de l'extérieur vers l'intérieur.

  Pour écrire le code~: \pyret{first-n-rows(}~$\rightarrow$~\pyret{sort(resultats, "edit-distance", true)}~$\rightarrow$~\pyret{, 5)}
\end{keypointbox}

\subsection{Composer dans l'autre sens}

\begin{thinkbox}
  Que se passe-t-il si on inverse l'ordre~? \pyret{sort(first-n-rows(resultats, 5), "edit-distance", true)} — est-ce que ça donne le même résultat que \pyret{first-n-rows(sort(resultats, "edit-distance", true), 5)}~?
  \begin{itemize}
    \item \textbf{Non.} Le premier trie seulement 5 lignes (inutile), le second extrait les 5 meilleures après avoir tout trié (utile~!).
    \item \textbf{L'ordre de composition compte.}
  \end{itemize}
\end{thinkbox}

\subsection{Composer avec \pyret{count} et \pyret{bar-chart}}

Même principe pour visualiser~: on ne passe pas la table brute à \pyret{bar-chart}, on passe d'abord par \pyret{count} pour obtenir les fréquences, \emph{puis} on visualise.

\begin{center}
  \pyret{bar-chart(count(resultats, "edit-distance"), "edit-distance", "count")}

  \vspace{0.3em}
  $\uparrow$ extérieur \hspace{4em} $\uparrow$ intérieur \hspace{3em} $\uparrow$ colonnes
\end{center}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Écris le code pour obtenir un \textbf{diagramme circulaire} des 10 premiers résultats triés par distance décroissante.
    \item Dessine le Cercle d'Évaluation correspondant (combien de cercles emboîtés~?).
  \end{enumerate}
\end{activitybox}

\section{Interface utilisateur pour notre correcteur}

\begin{overviewbox}
  On découvre comment extraire des valeurs d'une table, mesurer des chaînes, et créer des images avec Pyret — pour construire une interface qui affiche les suggestions orthographiques.
\end{overviewbox}

\subsection{Accéder aux lignes~: \pyret{row-n}}

\contrat{row-n :: Table, Number -> Row}

\begin{codeexample}
  \begin{itemize}[leftmargin=*]
    \item \pyret{row-n(resultats, 0)["word"]} \hspace{2em} — premier mot suggéré
    \item \pyret{row-n(resultats, 0)["edit-distance"]} \hspace{2em} — sa distance
    \item \pyret{row-n(resultats, 1)["word"]} \hspace{2em} — deuxième mot suggéré
  \end{itemize}
\end{codeexample}

\begin{keypointbox}
  \pyret{row-n(t, n)["colonne"]} s'appelle un \textbf{accesseur de ligne} (\emph{row accessor}). Il extrait la valeur d'une cellule spécifique.
\end{keypointbox}

\subsection{Mesurer des chaînes~: \pyret{string-length}}

\contrat{string-length :: String -> Number}

\begin{codeexample}
  \pyret{string-length("bonjour")} \hspace{2em} $\rightarrow$ \texttt{7}

  \pyret{string-length(row-n(resultats, 0)["word"])} \hspace{2em} — longueur du 1\textsuperscript{er} mot
\end{codeexample}

\subsection{Dessiner du texte~: \pyret{text}}

\contrat{text :: String, Number, String -> Image}

\begin{codeexample}
  \pyret{text("Bonjour !", 50, "blue")} \hspace{2em} — texte bleu de taille 50
\end{codeexample}

\subsection{Fonctions d'images}

Plusieurs fonctions produisent des \pyret{Image}~:

\begin{center}
\begin{tabular}{ll}
  \toprule
  \textbf{Contrat} & \textbf{Description} \\
  \midrule
  \pyret{triangle :: Number, String, String -> Image} & Triangle (taille, type, couleur) \\
  \pyret{circle :: Number, String, String -> Image}   & Cercle (rayon, type, couleur) \\
  \pyret{star :: Number, String, String -> Image}     & Étoile (rayon, type, couleur) \\
  \pyret{rectangle :: Number, Number, String, String -> Image} & Rectangle (largeur, hauteur, type, couleur) \\
  \pyret{ellipse :: Number, Number, String, String -> Image}  & Ellipse (largeur, hauteur, type, couleur) \\
  \pyret{regular-polygon :: Number, Number, String, String -> Image} & Polygone régulier (nb~côtés, rayon, type, couleur) \\
  \pyret{overlay :: Image, Image -> Image} & Superpose deux images \\
  \pyret{rotate :: Number, Image -> Image} & Fait pivoter une image (degrés) \\
  \bottomrule
\end{tabular}
\end{center}

\subsection{Construire une interface — Cercle d'Évaluation}

\begin{activitybox}
  \textbf{Objectif}~: superposer le mot \pyret{"Platoon"} (noir, taille 20) au-dessus d'un rectangle gris plein de 150×50.

  \begin{enumerate}[leftmargin=*]
    \item Dessine le Cercle d'Évaluation (fonction extérieure~: \pyret{overlay})
    \item Convertis-le en code~:\\
      \pyret{overlay(}\\
      \ \ \pyret{text("Platoon", 20, "black"),}\\
      \ \ \pyret{rectangle(150, 50, "solid", "gray"))}
    \item \textbf{Sois créatif·ve~!} Ajoute de la couleur, plusieurs formes, voire plusieurs suggestions~!
  \end{enumerate}
\end{activitybox}

\section{Visualiser les données}

\subsection{Table de fréquences~: \pyret{count}}

\contrat{count :: Table, String -> Table}

\begin{codeexample}
  \pyret{count(resultats, "edit-distance")}
\end{codeexample}

\pyret{count} produit une table de fréquences~: combien de suggestions pour chaque distance.

\subsection{Diagramme en barres~: \pyret{bar-chart}}

\contrat{bar-chart :: Table, String -> Image}

\begin{codeexample}
  \pyret{bar-chart(count(resultats, "edit-distance"), "edit-distance", "count")}
\end{codeexample}

\begin{thinkbox}
  \begin{itemize}
    \item Passe le curseur sur une barre. Quelle information obtiens-tu~?
    \item Essaie \pyret{bar-chart} avec la colonne \pyret{"word"}. Est-ce utile~? Pourquoi~?
  \end{itemize}
\end{thinkbox}

\subsection{Diagramme circulaire~: \pyret{pie-chart}}

\contrat{pie-chart :: Table, String -> Image}

\begin{codeexample}
  \pyret{pie-chart(resultats, "edit-distance")}
\end{codeexample}

\subsection{Les vrais correcteurs renvoient des mots, pas des tables~!}

\begin{activitybox}
  \begin{enumerate}[leftmargin=*]
    \item Dans la Zone de Définitions, ajoute~: \pyret{ligne2 = row-n(resultats, 1)}. Clique «~Run~».
    \item Évalue \pyret{ligne2}. Qu'obtiens-tu~?
    \item Évalue \pyret{ligne2["word"]} et \pyret{ligne2["edit-distance"]}.
    \item Comment obtenir la distance d'édition du 5\textsuperscript{e} mot suggéré~?
  \end{enumerate}
\end{activitybox}

\section{Synthèse}

\begin{thinkbox}
  \begin{itemize}
    \item \pyret{first-n-rows} et \pyret{last-n-rows} ont le même Domaine et la même Image, mais font des choses différentes. Comment est-ce possible~?
    \begin{itemize}
      \item Ce dont une fonction a \textbf{besoin} $\neq$ ce qu'elle \textbf{fait}.
    \end{itemize}
    \item Pourquoi \pyret{sort} a-t-il besoin de plus d'arguments que les autres~?
    \item Pourquoi un correcteur a-t-il besoin d'accesseurs de ligne~?
    \begin{itemize}
      \item On veut suggérer un \emph{mot}, pas une ligne entière~!
    \end{itemize}
  \end{itemize}
\end{thinkbox}
